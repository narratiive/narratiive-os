NARRATIIVE VISUAL INTELLIGENCE SYSTEM v1
PURPOSE
The purpose of this document is to ensure that every Narratiive Blueprint feels like the work of a senior strategist and a creative director working together.
Most strategy decks fail because they communicate insight through text.
Narratiive communicates insight through visual thinking.
The objective is not to fill slides.
The objective is to create moments of understanding.
The reader should repeatedly think:
* “I hadn’t considered that.”
* “I’ve never seen the category framed this way.”
* “This changes how I think about growth.”
* “This is worth acting on.”
Every slide should reveal something, not merely describe it.
________________


CORE PHILOSOPHY
Poor strategy decks explain.
Great strategy decks reveal.
Every slide should follow:
Insight
↓
Evidence
↓
Visualisation
↓
Implication
↓
Action
Visual communication is always preferred to written explanation.
The visual should communicate the core idea within five seconds.
If the visual cannot be described clearly, the thinking is not clear enough.
________________


SLIDE CONSTRUCTION RULES
Every slide must contain:
1. Strategic Insight
2. Recommended Visual
3. Supporting Evidence
4. Strategic Implication
No slide should rely solely on paragraphs.
Maximum body copy:
100 words.
Maximum paragraph length:
50 words.
Maximum bullet count:
5 bullets.
If a slide exceeds these limits, redesign the slide.
________________


MANDATORY SLIDE COMPONENTS
Every slide must contain at least one of the following:
Strategic Framework
A visual model explaining relationships between concepts.
Examples:
* Growth Engine
* Narrative Engine
* Attention Flywheel
* Consumer Decision Model
* Category Tension Framework
________________


Data Visualisation
A chart, graph, scoring system, ranking or quantified comparison.
Examples:
* Market growth
* Category penetration
* Competitive share
* Consumer trends
* Opportunity scoring
________________


Strategic Comparison
A comparison between two competing realities.
Examples:
Current State vs Future State
Consumer Need vs Category Delivery
Market Assumption vs Market Reality
Traditional Behaviour vs Emerging Behaviour
________________


Decision Matrix
A prioritisation framework.
Examples:
Impact vs Effort
Market Attractiveness vs Competitive Density
Commercial Value vs Strategic Fit
Opportunity Size vs Ease of Execution
________________


FOUNDER INSIGHT BOXES
Every Blueprint should contain 3–5 Founder Insight Boxes.
Purpose:
Challenge conventional thinking.
Create moments of surprise.
Format:
MOST FOUNDERS BELIEVE
[Common assumption]
THE EVIDENCE SUGGESTS
[Counterpoint]
Example:
Most founders believe growth comes from loyalty.
The evidence suggests growth primarily comes from reaching more buyers.
Founder Insight Boxes should feel memorable enough to repeat in conversation.
________________


THE “SO WHAT?” RULE
Every slide must answer:
SO WHAT?
The answer should be one sentence.
Example:
SO WHAT?
Maeving should compete against urban friction, not against other motorcycles.
If the slide cannot answer “So What?” clearly, the slide is incomplete.

DATA VISUALISATION RULES
Data should support insight.
Data should never exist without interpretation.
Every chart must answer:
1. What does this show?
2. Why does it matter?
3. What should happen next?
Avoid:
* excessive data
* complex tables
* spreadsheet exports
Prefer:
* rankings
* comparisons
* trend lines
* scoring systems
* benchmark comparisons

QUALITY CONTROL CHECKLIST
Before finalising any Blueprint ask:
Does every slide contain a visual framework?
Does every slide contain a clear insight?
Does every slide answer “So What?”
Would a founder find this useful?
Would this challenge conventional thinking?
Would this feel premium?
Would this feel worth £5,000?
Would the client screenshot this slide?
Would the client share this slide internally?
Would the client remember this slide a week later?
If the answer is no, improve the slide.
